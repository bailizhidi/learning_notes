弯管仿真数据#

数据集计划#

首先管子的长度以及弯曲位置，对弯曲和回弹的状态没有影响，可以不考虑。

其次是几何方面：弯曲半径，管子厚度和直径，三个变量。

  • 设定参数边界与物理约束确定采样范围：外径 \(D\)：[6.0 mm, 12.0 mm]厚度 \(t\)：[0.4 mm, 1.0 mm]弯曲半径 \(R\)：[18.0 mm, 32.0 mm]

  • 设定物理剔除规则 (剔除在 Abaqus 中极难收敛的极端情况)：相对弯曲半径约束：设定 \(R/D \ge 2.0\)。弯曲半径太小会导致外壁严重拉裂。相对壁厚约束：设定 \(t/D \ge 0.05\)。管壁太薄会导致内侧严重起皱失稳。

  • 使用 Python 生成采样矩阵，生成 100 个仿真输入参数，使用拉丁超立方抽样（LHS）结合物理约束过滤来生成参数矩阵。

弯曲阶段弯曲角度都取90度，然后弯曲仿真输出90帧，采用等时间间隔输出，每帧变化1度，回弹输出两帧，弯曲结束帧和回弹结束帧。弯曲阶段用MeshGraphNets训练，回弹阶段用线性映射，输出各种场数据。

Abaqus弯管数据集提取详细内容：#

1.提取的数据字段结构#

提取脚本将多部件装配体（管身 + 4个离散刚体模具）的历史输出映射到了全局节点数组中。生成的 .npz 文件包含以下 8 个核心 Numpy 数组：

数组键名 (Key)

维度 (Shape)

数据类型

物理含义与提取逻辑

mesh_pos

\([N, 3]\)

float32

初始空间坐标。即 \(t=0\) 帧时所有节点在全局坐标系下的绝对 \(X, Y, Z\) 坐标。

world_pos

\([T, N, 3]\)

float32

瞬态空间坐标。由 Abaqus 的位移场 U 与初始坐标相加得到。记录了 \(T\) 个时间步内所有节点的运动轨迹。

cells

\([E, 4]\)

int32

网格拓扑连接关系。提取了 S4R 壳单元的四节点连接索引，用于后续构建图结构的物理边。

node_type

\([N, 1]\)

int32

物理边界条件标签。
• 0: 自由管身（需要 GNN 预测位移和应力）
• 1: 模具障碍物（只提供碰撞约束）
• 3: 夹紧驱动端（利用 \(Z \le 0.1\) 切割获得，强制驱动管子运动）

thickness

\([N, 1]\)

float32

管壁厚度特征。由于壳单元在几何上是 0 厚度的二维面，该数组从 DOE 文件夹名称中解析厚度值（模具节点赋为 0.0），赋予管子真实的截面抗弯刚度信息。

stress

\([T, N, 1]\)

float32

米塞斯等效应力 (Von Mises Stress)。利用 ELEMENT_NODAL 提取，并遍历壳单元厚度方向的所有截面层，保留最危险的最大应力极值。

strain

\([T, N, 1]\)

float32

对数应变 (Max Principal LE)。同上，捕捉壳体变形弯曲时的极限主应变。

peeq

\([T, N, 1]\)

float32

等效塑性应变 (PEEQ)。标识节点是否进入屈服阶段及塑性损伤程度。

(注：\(N\) 为全局总节点数，\(T\) 为总帧数，\(E\) 为管身的单元总数。)

2.与 MeshGraphNets 架构的映射关系#

MeshGraphNets 本质上是一个“编码器-处理器-解码器 (Encoder-Processor-Decoder)” 架构。提取的数据将通过以下方式驱动 GNN 完成大变形接触物理的预测：

A. 构建图结构 (Graph Construction)

  • Mesh Edges (网格边)：由 cells 数组构建。它告诉 GNN 哪些节点在材料内部是物理相连的。应力正是通过这些边像多米诺骨牌一样在管身（Type 0 和 3）内传递的。

  • World Edges (世界边 / 碰撞边)：由 world_pos 数组在当前帧的距离动态计算得出（Radius Search）。当自由管身（Type 0）靠近压模/弯模（Type 1）时，动态生成世界边，传递接触排斥力，从而防止穿模。

B. 节点特征输入 (Node Features / Encoder Input) 在进入 GNN 的 Encoder 之前，需要把以下数据拼接成节点特征向量：

  1. 节点类型 (Node Type)：将 node_type 转换为 One-Hot 向量（如 [1,0,0], [0,1,0], [0,0,1]），告诉网络该节点当前的“角色”（是被拉扯、是墙壁，还是需要自己动）。

  2. 历史速度 (Velocity)：利用 world_pos[t] - world_pos[t-1] 计算得到。

  3. 定制化物理属性 (Thickness)：直接拼接 thickness 数组。这是网络能够泛化不同管壁厚度的核心输入特征！

C. 边界条件驱动 (Kinematic Boundary)

  • 在预测 (Rollout) 阶段，world_pos 中的 Type 3 (夹紧端) 节点的轨迹不仅是真实答案，更是驱动引擎。预测代码将强行把数学算出的旋转坐标覆盖给 Type 3 节点，网络据此计算应变，并向后拖拽 Type 0 节点。

D. 网络预测目标 (Target / Decoder Output) GNN 的 Decoder 的输出维度通常与你需要预测的物理场对齐。在监督学习中，网络输出将与以下提取数据计算 Loss (均方误差 MSE)：

  1. 目标速度/加速度：利用 world_pos[t+1] 计算，使得管身能随着模具的挤压和拉扯正确向前运动并弯曲。

  2. 目标物理场 (Fields)：直接对应提取出的 stress, strain, peeq 数组。GNN 需要在仅仅知道节点位移的情况下，精准推断出管身内部达到了上千兆帕的应力极值。

3.本数据集工程的特殊高光点#

  • 智能厚度注入 (Shell Element Thickness Injection)：完美解决了壳单元零厚度导致的几何特征丢失问题，大幅降低了 3D 实体网格计算的巨大算力开销。

  • 截面点极值剥离 (Section Point Max-Pooling)：通过 Abaqus 的 ELEMENT_NODAL API，绕过了 CAE 界面的渲染平滑滤镜，提取了壳体在弯曲过程中受拉/受压两侧的绝对极限物理响应，为深度学习提供了最锐利、最纯粹的梯度信号。

  • 刚体全网格化 (Discrete Rigid Meshing)：模具全面离散化（赋予 Type 1），使得 MeshGraphNet 的强项——空间接触动态建图 (Dynamic Radius Search) 得以完美施展。

训练计划#

单样本测试#

  • v1：强迫网络用 10 倍的精力去把几何形状算准，防止炸管total_loss = 10.0 * loss_vel + 1.0 * loss_stress + 1.0 * loss_strain + 0.1 * loss_peeq（四号卡）（结果较好）

  • v2：物理平滑约束：保持位移主导的同时，给应力和应变赋予足够的“熨平”能力total_loss = 10.0 * loss_vel + 5.0 * loss_stress + 5.0 * loss_strain + 2.0 * loss_peeq（五号卡）

  • v3:total_loss = 10.0 * loss_vel + 5.0 * loss_stress + 5.0 * loss_strain + 2.0 * loss_peeq + 100.0 * loss_edge_penalty加入位移偏差过大惩罚，更加平滑（三号卡）

多样本测试（10组数据）#

对于多样本训练，10组数据，8组为训练集，2组为测试集合，其中一组为内插一组为外插

  • v1：噪声加大到0.1，loss采用total_loss = 10.0 * loss_vel + 1.0 * loss_stress + 1.0 * loss_strain + 0.1 * loss_peeq，效果良好

  • v2：lr: 0.00001，noise_std: 0.01，total_loss = 10.0 * loss_vel + 5.0 * loss_stress + 1.0 * loss_strain + 0.1 * loss_peeq，有鼓包，loss下降慢

  • v3：lr: 0.00005，total_loss = 10.0 * loss_vel + 2.0 * loss_stress + 1.0 * loss_strain + 0.1 * loss_peeq，效果不好

  • v4：lr：0.0001,noise_std: 0.1，total_loss = 10.0 * loss_vel + 2.0 * loss_stress，只预测位移和应变，500个epoch后，效果稍有改进，但是管子弯曲处还是有凹陷和鼓包，接下来降低噪声，继续训练

多样本测试（27组数据）#

27组数据，24组作为训练集合，3组作为测试集合，一组内插，一组边界极端，一组外插，包含各类场景

  • v1：噪声取0.05，前400轮lr：0.0001，后200轮lr：0.00001，共计600个epoch，total_loss = 10.0 * loss_vel + 2.0 * loss_stress
