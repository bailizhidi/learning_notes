# 弯管仿真数据
## 数据集计划
首先管子的长度以及弯曲位置，对弯曲和回弹的状态没有影响，可以不考虑。

其次是几何方面：弯曲半径，管子厚度和直径，三个变量。

- 设定参数边界与物理约束确定采样范围：外径 $D$：[6.0 mm, 12.0 mm]厚度 $t$：[0.4 mm, 1.0 mm]弯曲半径 $R$：[18.0 mm, 32.0 mm]
- 设定物理剔除规则 (剔除在 Abaqus 中极难收敛的极端情况)：相对弯曲半径约束：设定 $R/D \ge 2.0$。弯曲半径太小会导致外壁严重拉裂。相对壁厚约束：设定 $t/D \ge 0.05$。管壁太薄会导致内侧严重起皱失稳。
- 使用 Python 生成采样矩阵，生成 100 个仿真输入参数，使用拉丁超立方抽样（LHS）结合物理约束过滤来生成参数矩阵。

弯曲阶段弯曲角度都取90度，然后弯曲仿真输出90帧，采用等时间间隔输出，每帧变化1度，回弹输出两帧，弯曲结束帧和回弹结束帧。弯曲阶段用MeshGraphNets训练，回弹阶段用线性映射，输出各种场数据。

关于仿真方面，模具的建模需要采用离散刚体，因为需要对其划分网格，判断接触点

节点类型与物理角色对应关系：
| 部件名称 (Instance) | 物理角色 | 节点类型标签 (Node Type) | GNN 负责做什么？ |
|---|---|---|---|
| TUBE-1 (主体管身) | 自由变形区 | 0 (Moving / Free) | 核心预测对象。GNN 预测这里的位移和应力，它会和模具发生碰撞。 |
| TUBE-1 (被夹紧的前端) | 强制位移区 | 3 (Clamped / Kinematic) | 边界驱动力。GNN 不预测它，在代码里直接给它赋真实坐标，靠它拉扯管身。 |
| YAMU-1 (压模) | 障碍物 / 碰撞壁 | 1 (Object / Obstacle) | 提供接触约束。GNN 不预测它，它只需呆在原地或随动，管子碰到它就会被弹开。 |
| RAOWANMU-1 (弯模) | 障碍物 / 碰撞壁 | 1 (Object / Obstacle) | 同上。 |
| XINBANG-1 (芯棒) | 障碍物 / 碰撞壁 | 1 (Object / Obstacle) | 同上。 |
| XUANZHUANMU-1 (夹紧模) | 障碍物 / 碰撞壁 | 1 (Object / Obstacle) | 同上。其实管端（类型3）就是被它咬住的，它提供了接触包裹。 |
